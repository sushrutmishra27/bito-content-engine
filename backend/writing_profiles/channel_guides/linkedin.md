# LinkedIn Content Guide

## Format
- 800-1500 characters per post
- Short paragraphs (1-2 lines)
- Use line breaks liberally
- Hook in the first line (visible before "see more")

## Best Performing Patterns
1. **Contrarian take**: Challenge a popular belief with data/experience
2. **Behind the scenes**: Share real decisions, mistakes, learnings from building
3. **How-to with depth**: Teach something specific and actionable
4. **Data stories**: Share interesting metrics or findings
5. **Industry commentary**: React to news/trends with a unique angle

## Posting Times
- Best: Tuesday-Thursday, 8-10am EST
- Good: Monday/Friday, 8-10am EST
- Avoid: Weekends, evenings

## Hook Types
- Question hook: "Why do 80% of code reviews miss the bugs that matter?"
- Story hook: "Last week, we caught a bug that would have cost $50K."
- Contrarian hook: "Hot take: most AI code review tools are solving the wrong problem."
- Data hook: "We analyzed 10,000 PRs. Here's what we found."
- Challenge hook: "Your code review process is broken. Here's how to fix it."
